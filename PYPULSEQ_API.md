# PyPulseq 1.5.0.post1 — public API reference

Complete inventory of what [`pypulseq`](https://github.com/imr-framework/pypulseq) exposes at the top level of the `pypulseq` package in tag `v1.5.0.post1`. Generated by inspecting the installed `pypulseq/__init__.py` plus `inspect.signature` / `inspect.getdoc` on each exported object.

Use this as the reference surface pulseq-zero has to cover. Items marked **[new in 1.5]** were not present in 1.4.2 and are therefore not yet represented in the current pulseq-zero facade ([src/pulseqzero/\_\_init\_\_.py](src/pulseqzero/__init__.py)).

## Module-level constants / helpers

| Name               | Kind     | Description                                                           |
| ------------------ | -------- | --------------------------------------------------------------------- |
| `__version__`      | str      | Package version (via `importlib.metadata`).                           |
| `round_half_up`    | function | `round_half_up(n, decimals=0)` — half-up rounding (banker's-round workaround). |
| `eps`              | float    | Pulseq precision constant (`1e-9` for float64).                       |

## Classes

### `Sequence`

```py
Sequence(system: Opts | None = None, use_block_cache: bool = True)
```

Main entry point; holds an in-memory sequence definition and is what `.seq` files serialize. See the *`Sequence` methods* table below for its full surface.

### `Opts`

```py
Opts(
    adc_dead_time: float | None = None,
    adc_raster_time: float | None = None,
    block_duration_raster: float | None = None,
    gamma: float | None = None,
    grad_raster_time: float | None = None,
    grad_unit: str = 'Hz/m',
    max_grad: float | None = None,
    max_slew: float | None = None,
    rf_dead_time: float | None = None,
    rf_raster_time: float | None = None,
    rf_ringdown_time: float | None = None,
    adc_samples_limit: int | None = None,     # [new in 1.5]
    adc_samples_divisor: int | None = None,   # [new in 1.5]
    rise_time: float | None = None,
    slew_unit: str = 'Hz/m/s',
    B0: float | None = None,                   # [new in 1.5]
)
```

Scanner system limits. One instance method: `Opts.set_as_default()`.

### `SigpyPulseOpts`

```py
SigpyPulseOpts(
    pulse_type='slr', ptype='st', ftype='ls',
    d1=0.01, d2=0.01, cancel_alpha_phs=False,
    n_bands=3, band_sep=20, phs_0_pt='None',
)
```

Options bundle consumed by sigpy-backed pulse designers. The pulse-design functions (`sigpy_n_seq`, `make_slr`, `make_sms`) that used to be re-exported in 1.4.2 are no longer at the top level in 1.5.

## Delta vs PyPulseq 1.4.2 (summary)

For porting pulseq-zero to the 1.5 API, these are the deltas worth tracking:

**Removed from top-level re-exports** (vs 1.4.2):

- `sigpy_n_seq`, `make_slr`, `make_sms` — sigpy-backed pulse designers. Already commented out in [src/pulseqzero/\_\_init\_\_.py](src/pulseqzero/__init__.py) (the 1.4.2.post1 drop is the same).
- `calc_SAR` is still re-exported but is now a placeholder that raises `DeprecationError` when called.

**Added in 1.5 (top-level)**:

- `calc_adc_segments`
- `make_soft_delay`
- `enable_trace`, `disable_trace`

**Added in 1.5 (Sequence methods)**:

- `apply_soft_delay`
- `find_block_by_time`
- `paper_plot`
- `register_soft_delay_event`

**New kwargs on existing factories** (additive — old calls still work):

- `freq_ppm`, `phase_ppm` on `make_sinc_pulse`, `make_gauss_pulse`, `make_block_pulse`, `make_arbitrary_rf`, `make_adiabatic_pulse`, `make_adc`.
- `center` on `make_arbitrary_rf`.
- `phase_modulation` on `make_adc`.
- `oversampling` on `make_arbitrary_grad` and `calc_ramp`.

**New `Opts` fields**:

- `adc_samples_limit`, `adc_samples_divisor` — Siemens-style ADC segmentation constraints.
- `B0` — main-field strength (used in off-resonance / ppm conversions).

**Backward-incompatible defaults to watch for**:

- `make_sinc_pulse`, `make_gauss_pulse`, `make_block_pulse`, `make_arbitrary_rf`, `make_adiabatic_pulse` now default `use='undefined'` (was `use=''` in 1.4.2 and pulseq-zero's adapter). The adapter still accepts `''`; the pypulseq-level write path should map `''` → `'undefined'` or vice-versa to stay consistent.
